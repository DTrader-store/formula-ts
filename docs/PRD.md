## 1. 产品概述

### 1.1 产品定义

**Formula-TS** 是一个用 TypeScript 编写的通达信公式解析器和执行引擎，为开发者和量化交易者提供在 Web 应用中解析和执行通达信技术指标公式的能力。

### 1.2 产品愿景

成为金融技术领域最易用、高性能的技术指标公式解析引擎，让开发者能够轻松地在自己的应用中集成强大的技术分析功能。

### 1.3 核心价值主张

- **兼容性**: 完全兼容通达信公式语法，降低用户学习成本
- **高性能**: 支持大规模数据计算（10000+ 数据点），增量计算延迟 < 100ms
- **同构设计**: 同时支持浏览器和 Node.js 环境
- **易于集成**: 简洁的 API 设计，开箱即用
- **可扩展性**: 灵活的函数注册机制，易于添加自定义函数

## 3. 功能需求

### 3.1 核心功能（MVP - Phase 1）

#### F1: 公式解析
**优先级**: P0 (必须有)

**功能描述**:
- 词法分析：将公式文本转换为 Token 流
- 语法分析：构建抽象语法树 (AST)
- 支持通达信基础语法：
  - 变量声明（`:=`）
  - 输出声明（`:`）
  - 算术运算符（`+`, `-`, `*`, `/`）
  - 比较运算符（`>`, `<`, `>=`, `<=`, `=`, `<>`）
  - 逻辑运算符（`AND`, `OR`）
  - 函数调用
  - 括号表达式

**验收标准**:
- [ ] 能正确解析 MA、MACD、KDJ 等经典指标公式
- [ ] 对于语法错误，能准确定位并给出清晰的错误信息（行号、列号）
- [ ] 单元测试覆盖率 > 80%

#### F2: 公式执行
**优先级**: P0 (必须有)

**功能描述**:
- 遍历 AST 并执行计算
- 支持市场数据变量：OPEN、CLOSE、HIGH、LOW、VOLUME、AMOUNT
- 实现 10 个核心函数：
  - 数学函数：MA（移动平均）、EMA（指数移动平均）、SUM（求和）
  - 引用函数：REF（引用）、HHV（最高值）、LLV（最低值）
  - 逻辑函数：IF（条件）、CROSS（交叉）
  - 工具函数：MAX（最大值）、MIN（最小值）

**验收标准**:
- [ ] 计算结果与通达信官方结果一致（误差 < 0.01%）
- [ ] 支持至少 1000 个数据点的计算
- [ ] 每个函数都有完整的单元测试

#### F3: 基础错误处理
**优先级**: P0 (必须有)

**功能描述**:
- 定义错误类型：LexerError、ParserError、RuntimeError
- 错误信息包含：错误位置、错误描述、修复建议
- 优雅的错误传播机制

**验收标准**:
- [ ] 所有可预见的错误都有明确的错误类型
- [ ] 错误信息对用户友好，易于理解
- [ ] 错误处理不使用异常控制正常流程

#### F4: 基础 API
**优先级**: P0 (必须有)

**功能描述**:
```typescript
// 主入口类 FormulaEngine
const engine = new FormulaEngine();

// 编译公式
const compiled = engine.compile(formulaText);

// 执行计算
const result = compiled.execute(marketData);

// 结果格式
interface FormulaResult {
  outputs: {
    name: string;      // 输出线名称，如 "MA5"
    values: number[];  // 计算结果数组
    style?: {          // 绘图样式
      color?: string;
      lineWidth?: number;
      lineType?: 'solid' | 'dashed' | 'stick';
    };
  }[];
  errors?: Error[];
}
```

**验收标准**:
- [ ] API 设计简洁，文档完整
- [ ] 所有公共接口都有 TypeScript 类型定义
- [ ] 提供完整的 JSDoc 注释

- 无已知安全漏洞

## 5. 技术要求

### 5.1 技术栈

**核心**:
- TypeScript >= 4.5
- 无运行时依赖（纯 TypeScript 实现）

**开发工具**:
- Jest - 测试框架
- ESLint + Prettier - 代码质量
- TypeDoc - 文档生成
- Vite 或 Rollup - 构建工具

**演示页面**:
- TradingView Lightweight Charts - 图表库

### 5.2 架构设计

**整体架构**:
```
公式文本 → Lexer → Tokens → Parser → AST → Interpreter → 结果
                                              ↓
                                      FunctionRegistry
                                              ↓
                                        ChartFormatter
```

**设计模式**:
- **编译器模式**: Lexer → Parser → AST → Interpreter
- **Visitor 模式**: AST 遍历和执行
- **Registry 模式**: 函数管理
- **Strategy 模式**: 输出格式化

**模块划分**:
```
src/
├── lexer/           - 词法分析
├── parser/          - 语法分析
├── interpreter/     - 解释执行
├── formatter/       - 输出格式化
├── functions/       - 内置函数库
├── types/           - 类型定义
└── utils/           - 工具函数
```

### 5.3 代码规范

**命名约定**:
- 类名: PascalCase
- 方法名: camelCase
- 常量: UPPER_SNAKE_CASE
- 接口: PascalCase（可选 `I` 前缀）

**TypeScript 规范**:
- 严格模式: `strict: true`
- 显式类型注解
- 避免使用 `any`
- 优先使用接口定义数据结构

## 6. 约束和限制

### 6.1 功能范围限制（第一版本）

**仅支持**:
- 技术指标公式（不支持选股、交易系统公式）
- 20-30 个核心函数
- 单周期数据（不支持跨周期引用）

**不支持**:
- 自定义函数定义（使用内置函数）
- 跨周期引用（如日线引用周线数据）
- 数据获取功能（用户需自行提供数据）
- 公式语法自动修复

### 6.2 数据依赖

- 用户必须提供市场数据（OHLCV 格式）
- 不内置数据源
- 数据格式和质量由用户保证

### 6.3 错误恢复策略

- 语法错误直接报错，不尝试修复
- 运行时错误停止计算，返回错误信息
- 不进行错误容错（fail-fast 原则）
