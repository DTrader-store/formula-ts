# 完整 Lexer 实现（词法分析器）

**日期**: 2025-11-14

## 问题描述

需要实现一个完整的词法分析器（Lexer），用于将通达信公式源代码转换为 Token 序列。这是公式解析器的第一个核心组件，必须支持：

- 数字识别（整数和浮点数）
- 标识符和关键字识别
- 运算符识别
- 分隔符识别
- 注释处理
- 精确的位置跟踪（行号和列号）
- 完善的错误处理

## 分析过程

### 技术要求分析

根据 `openspec/project.md` 中的通达信公式语法规范，Lexer 需要处理：

1. **数字类型**
   - 整数：`123`, `456`
   - 浮点数：`123.456`, `.5`
   - 需要支持以小数点开头的数字

2. **标识符和关键字**
   - 市场数据变量：`OPEN`, `CLOSE`, `HIGH`, `LOW`, `VOLUME`, `AMOUNT`
   - 函数名：`MA`, `EMA`, `SMA`, `HHV`, `LLV` 等
   - 用户定义变量：`MA5`, `VAR1`, `K`, `D`, `J` 等
   - 关键字：`IF`, `AND`, `OR`
   - 绘图属性：`COLORRED`, `COLORGREEN`, `LINETHICK1`-`LINETHICK9`, `DOTLINE`, `STICK`

3. **运算符**
   - 算术运算符：`+`, `-`, `*`, `/`
   - 比较运算符：`>`, `<`, `>=`, `<=`, `=`, `<>`
   - 赋值运算符：`:=`

4. **分隔符**
   - 括号：`(`, `)`
   - 逗号：`,`
   - 分号：`;`
   - 冒号：`:`

5. **注释**
   - 单行注释：`// comment`
   - 块注释：`{ comment }`

### 设计决策

1. **TDD 方法**：先编写完整的测试套件，再实现功能，确保所有边界情况都被覆盖

2. **位置跟踪**：维护 `line` 和 `column` 计数器，便于后续错误报告

3. **关键字识别策略**：
   - 使用 Map 存储固定关键字（`IF`, `AND`, `OR` 等）
   - 使用模式匹配识别 `COLOR*` 和 `LINETHICK*` 系列

4. **空白符处理**：
   - 普通空格和制表符直接跳过
   - 连续换行符合并为单个 `NEWLINE` token
   - 支持 `\r\n` 和 `\n` 两种换行格式

5. **错误处理**：
   - 使用 `LexerError` 类提供详细的错误位置信息
   - 识别未闭合的块注释
   - 报告无效字符

## 解决方案

### 实现架构

```
Lexer
  ├── 核心方法
  │   └── tokenize(): Token[]          - 主入口，返回所有 tokens
  ├── 字符处理
  │   ├── advance()                    - 前进到下一字符
  │   ├── peek()                       - 查看下一字符（不消费）
  │   └── 位置跟踪（line, column）
  ├── 识别方法
  │   ├── readNumber()                 - 识别数字
  │   ├── readIdentifierOrKeyword()    - 识别标识符/关键字
  │   └── readOperatorOrDelimiter()    - 识别运算符/分隔符
  └── 辅助方法
      ├── skipWhitespace()             - 跳过空白符
      ├── skipNewline()                - 处理换行
      ├── skipSingleLineComment()      - 跳过单行注释
      └── skipBlockComment()           - 跳过块注释
```

### 核心算法

#### 1. 主循环（tokenize）

```typescript
tokenize(): Token[] {
  const tokens: Token[] = [];
  let lastWasNewline = false;

  while (this.currentChar !== null) {
    // 跳过空白符
    if (this.isWhitespace(this.currentChar)) {
      this.skipWhitespace();
      continue;
    }

    // 处理换行（避免连续换行）
    if (this.isNewline(this.currentChar)) {
      if (!lastWasNewline) {
        tokens.push(this.makeToken(TokenType.NEWLINE, '\n'));
        lastWasNewline = true;
      }
      this.skipNewline();
      continue;
    }

    lastWasNewline = false;

    // 跳过注释
    if (this.currentChar === '/' && this.peek() === '/') {
      this.skipSingleLineComment();
      continue;
    }

    if (this.currentChar === '{') {
      this.skipBlockComment();
      continue;
    }

    // 识别 tokens
    if (this.isDigit(this.currentChar) ||
        (this.currentChar === '.' && this.isDigit(this.peek()))) {
      tokens.push(this.readNumber());
      continue;
    }

    if (this.isAlpha(this.currentChar)) {
      tokens.push(this.readIdentifierOrKeyword());
      continue;
    }

    const operatorToken = this.readOperatorOrDelimiter();
    if (operatorToken) {
      tokens.push(operatorToken);
      continue;
    }

    // 无效字符
    throw new LexerError(
      `Unexpected character: '${this.currentChar}'`,
      this.line,
      this.column,
      this.currentChar
    );
  }

  tokens.push(this.makeToken(TokenType.EOF, ''));
  return tokens;
}
```

#### 2. 数字识别

支持整数、浮点数和以小数点开头的数字：

```typescript
readNumber(): Token {
  const startColumn = this.column;
  let value = '';

  // 处理以小数点开头的数字
  if (this.currentChar === '.') {
    value += '.';
    this.advance();
  }

  // 读取整数部分
  while (this.currentChar !== null && this.isDigit(this.currentChar)) {
    value += this.currentChar;
    this.advance();
  }

  // 读取小数部分
  if (this.currentChar === '.' && value !== '.') {
    value += '.';
    this.advance();

    while (this.currentChar !== null && this.isDigit(this.currentChar)) {
      value += this.currentChar;
      this.advance();
    }
  }

  return new Token(TokenType.NUMBER, value, this.line, startColumn);
}
```

#### 3. 关键字识别

使用模式匹配和 Map 查找：

```typescript
readIdentifierOrKeyword(): Token {
  const startColumn = this.column;
  let value = '';

  // 读取字母数字字符
  while (this.currentChar !== null && this.isAlphaNumeric(this.currentChar)) {
    value += this.currentChar;
    this.advance();
  }

  const upperValue = value.toUpperCase();

  // 检查 COLOR* 关键字
  if (upperValue.startsWith('COLOR') && upperValue.length > 5) {
    return new Token(TokenType.COLOR, upperValue, this.line, startColumn);
  }

  // 检查 LINETHICK* 关键字
  if (upperValue.startsWith('LINETHICK') && upperValue.length === 10) {
    const thicknessChar = upperValue[9];
    if (thicknessChar >= '1' && thicknessChar <= '9') {
      return new Token(TokenType.LINETHICK, upperValue, this.line, startColumn);
    }
  }

  // 检查标准关键字
  const keywordType = this.keywords.get(upperValue);
  if (keywordType !== undefined) {
    return new Token(keywordType, upperValue, this.line, startColumn);
  }

  // 普通标识符
  return new Token(TokenType.IDENTIFIER, value, this.line, startColumn);
}
```

#### 4. 运算符识别

处理单字符和双字符运算符：

```typescript
readOperatorOrDelimiter(): Token | null {
  const startColumn = this.column;
  const char = this.currentChar!;

  switch (char) {
    case ':':
      this.advance();
      // 检查 :=
      if (this.currentChar === '=') {
        this.advance();
        return new Token(TokenType.ASSIGN, ':=', this.line, startColumn);
      }
      return new Token(TokenType.COLON, ':', this.line, startColumn);

    case '>':
      this.advance();
      // 检查 >=
      if (this.currentChar === '=') {
        this.advance();
        return new Token(TokenType.GTE, '>=', this.line, startColumn);
      }
      return new Token(TokenType.GT, '>', this.line, startColumn);

    case '<':
      this.advance();
      // 检查 <= 或 <>
      if (this.currentChar === '=') {
        this.advance();
        return new Token(TokenType.LTE, '<=', this.line, startColumn);
      } else if (this.currentChar === '>') {
        this.advance();
        return new Token(TokenType.NEQ, '<>', this.line, startColumn);
      }
      return new Token(TokenType.LT, '<', this.line, startColumn);

    // ... 其他运算符和分隔符
  }
}
```

#### 5. 注释处理

支持两种注释格式：

```typescript
// 单行注释：跳到行尾
skipSingleLineComment(): void {
  this.advance(); // 跳过第一个 /
  this.advance(); // 跳过第二个 /

  while (this.currentChar !== null && !this.isNewline(this.currentChar)) {
    this.advance();
  }
}

// 块注释：跳到 }
skipBlockComment(): void {
  const startLine = this.line;
  const startColumn = this.column;

  this.advance(); // 跳过 {

  while (this.currentChar !== null && this.currentChar !== '}') {
    if (this.isNewline(this.currentChar)) {
      this.skipNewline();
    } else {
      this.advance();
    }
  }

  // 检查未闭合注释
  if (this.currentChar === null) {
    throw new LexerError(
      'Unterminated block comment',
      startLine,
      startColumn,
      '{'
    );
  }

  this.advance(); // 跳过 }
}
```

3. **智能的换行处理**：
   - 多个连续换行符合并为单个 `NEWLINE` token
   - 避免生成过多的换行 token

4. **关键字识别的灵活性**：
   - 固定关键字使用 Map 快速查找
   - 可变关键字（COLOR*, LINETHICK*）使用模式匹配
   - 大小写不敏感（统一转为大写）

5. **完善的错误处理**：
   - 使用自定义 `LexerError` 类
   - 错误信息包含位置和上下文
   - 识别未闭合的块注释

### 性能考虑

- 单次遍历：O(n) 时间复杂度
- 最小内存分配：原地处理，只分配 Token 对象
- 关键字查找：O(1) Map 查找

### 已知限制

1. **不支持科学计数法**：如 `1.23e10`（通达信公式语法不需要）
2. **不支持负数字面量**：`-5` 会被识别为 `-` 和 `5` 两个 token（语义正确）
3. **不支持字符串字面量**：通达信公式不支持字符串

### 未来扩展

2. **更多关键字**：随着功能扩展可能需要添加更多关键字
3. **更丰富的错误提示**：提供修复建议
