## 细节与备注

### 设计决策

1. **多输出分离**: 选择为每个输出创建独立函数，而不是返回对象或数组
   - 优点: 符合通达信习惯，易于使用
   - 缺点: 函数数量增加

2. **NaN 处理**: 所有函数在数据不足时返回 NaN
   - 保持数组长度一致
   - 便于后续处理和图表显示

3. **参数验证**: 在函数内部进行基本验证
   - 检查数组长度
   - 处理边界情况

### 已知限制

1. **计算精度**: 浮点运算可能有微小误差
2. **性能**: 未使用 SIMD 或 WebAssembly 优化
3. **内存**: 每次调用都创建新数组，未使用对象池

### 未来改进

1. **性能优化**
   - 使用 ArrayBuffer 减少内存分配
   - 实现增量计算支持
   - 考虑 WebAssembly 实现
