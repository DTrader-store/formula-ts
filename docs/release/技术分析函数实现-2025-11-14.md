# 技术分析函数实现

**日期**: 2025-11-14

**Commit Hash**: 6840c37

## 问题描述

需要为公式解释器实现核心技术分析函数，以支持通达信风格的技术指标计算。这些函数是构建量化交易策略的基础组件。

需要实现以下技术分析函数：
- **SMA(X, N, M)**: 通达信特有的移动平均算法，带权重参数M
- **WMA(X, N)**: 线性加权移动平均
- **BOLL(X, N, P)**: 布林带，返回上、中、下轨三个数组
- **RSI(X, N)**: 相对强弱指标，范围0-100
- **ATR(N)**: 真实波幅，需要访问HIGH, LOW, CLOSE市场数据

### 2. 关键技术点

#### SMA函数的特殊性

通达信的SMA不是标准简单移动平均，而是加权平滑算法：

```
第一个值: (X[0] + X[1] + ... + X[N-1]) / N
后续值: (M * X[i] + (N-M) * SMA[i-1]) / N
```

- 当M=1时，给予历史值更多权重，产生平滑效果
- 当M=N时，完全使用当前值，无平滑
- 这是一种指数平滑的变体

#### BOLL函数的多值返回

BOLL需要返回三个数组（上轨、中轨、下轨），这在TypeScript中通过元组类型实现：

```typescript
export function BOLL(data: number[], N: number, P: number): [number[], number[], number[]]
```

注意：在函数注册表中暂时不注册BOLL，因为解释器需要特殊处理多值返回。

#### ATR函数的多参数输入

ATR需要三个输入数组（HIGH, LOW, CLOSE），计算真实波幅：

```
TR = max(
  High - Low,
  |High - Previous Close|,
  |Low - Previous Close|
)

ATR = EMA of TR
```

这需要从ExecutionContext获取市场数据，目前暂不在注册表中注册，等待解释器支持。

#### 2. WMA - 线性加权移动平均
```typescript
export function WMA(data: number[], N: number): number[] {
  // 权重: 1, 2, 3, ..., N (从旧到新)
  // WMA = (X[i-N+1]*1 + X[i-N+2]*2 + ... + X[i]*N) / (1+2+...+N)

```

**特点**:
- 线性递增权重，最新数据权重最大
- 权重和 = N*(N+1)/2

#### 3. BOLL - 布林带
```typescript

  // 上轨 = 中轨 + P * 标准差
  // 下轨 = 中轨 - P * 标准差

}
```

**特点**:
- 返回三个数组
- P参数控制波动带宽度（通常为2）
- 标准差计算使用总体标准差（除以N，而非N-1）

## 细节与备注

### 技术细节

1. **数组长度处理**
   - 所有函数返回与输入相同长度的数组
   - 数据不足时填充NaN
   - 确保结果数组可以直接与原始数据对齐

2. **数值精度**
   - 使用JavaScript原生number类型（IEEE 754双精度）
   - 测试中使用toBeCloseTo(value, 2)允许小数点后2位误差

3. **性能考虑**
   - 循环展开减少函数调用
   - 预计算常量（如权重和）
   - O(N*M)时间复杂度，N为数据长度，M为周期

### 未完成功能

1. **BOLL和ATR的注册表集成**
   - 原因：解释器暂不支持多值返回和多参数输入
   - 下一步：需要扩展解释器支持
   - 临时方案：函数可直接导入使用

2. **ExecutionContext集成**
   - ATR需要从context获取HIGH, LOW, CLOSE
   - 当前实现直接接受数组参数
   - 未来可能需要重载版本
