# 完整解释器实现

**日期**: 2025-11-14

## 问题描述

需要实现完整的解释器（Interpreter）以执行已解析的 AST（抽象语法树）。解释器需要：
- 管理执行上下文（市场数据、变量、输出）
- 使用访问者模式遍历和执行 AST 节点
- 支持所有表达式类型（二元、一元、函数调用等）
- 正确处理数组操作（所有计算都在数组上进行）
- 支持市场数据字段访问（OPEN, CLOSE, HIGH, LOW, VOLUME）

## 分析过程

### 1. 架构设计
解释器采用访问者模式（Visitor Pattern）来遍历 AST：
- **ExecutionContext**: 管理执行状态
- **Interpreter**: 实现访问者方法来执行各种节点

### 2. 关键设计决策

#### 2.1 数组为中心的计算模型
- 所有值都是 `number[]` 类型
- 字面量被扩展为与市场数据长度相同的数组
- 二元操作逐元素执行

#### 2.2 变量和输出管理
- 变量按声明顺序求值
- 输出可以引用之前声明的变量
- 使用 Map 存储变量和输出值

#### 2.3 函数调用处理
- 直接调用导入的函数实现（MA, EMA, SUM 等）
- 正确处理常量参数（如周期参数）
- 支持数组到数组的函数调用

visitProgram(program: Program): void {
  for (const statement of program.body) {
    this.visitStatement(statement);
  }
}

visitVariableDeclaration(node: VariableDeclaration): void {
  const value = this.visitExpression(node.value);
  this.context.setVariable(node.name, value);
}

visitOutputDeclaration(node: OutputDeclaration): void {
  const value = this.visitExpression(node.value);

#### 2.2 表达式求值

**二元表达式**:
```typescript
visitBinaryExpression(node: BinaryExpression): number[] {
  const left = this.visitExpression(node.left);
  const right = this.visitExpression(node.right);

  const length = Math.min(left.length, right.length);
  const result: number[] = new Array(length);

  for (let i = 0; i < length; i++) {
    result[i] = this.evaluateBinaryOperation(
      left[i],
      node.operator,
      right[i]
    );
  }

  return result;
}
```

**支持的运算符**:
- 算术: `+`, `-`, `*`, `/`, `%`, `^`
- 比较: `>`, `<`, `>=`, `<=`, `=`, `<>` （返回 1 或 0）
- 逻辑: `AND`, `OR` （非零值视为 true）

**一元表达式**:
```typescript
visitUnaryExpression(node: UnaryExpression): number[] {
  const operand = this.visitExpression(node.operand);
  const result: number[] = new Array(operand.length);

  for (let i = 0; i < operand.length; i++) {
    result[i] = this.evaluateUnaryOperation(node.operator, operand[i]);
  }

  return result;
}
```

**支持的运算符**: `-` (取反), `!` (逻辑非)

#### 2.3 函数调用

```typescript
visitFunctionCall(node: FunctionCall): number[] {
  const functionName = node.name.toUpperCase();
  const args = node.arguments.map((arg) => this.visitExpression(arg));

  switch (functionName) {
    case 'MA':
      const period = this.getConstantValue(args[1]);
      return MA(args[0], period);

    case 'MAX':
      return MAX(args[0], args[1]);

    case 'IF':
      return IF(args[0], args[1], args[2]);

    // ... 其他函数
  }
}
```

**支持的函数**: MA, EMA, SUM, MAX, MIN, REF, HHV, LLV, IF, CROSS

#### 2.4 标识符和字面量

```typescript
visitIdentifier(node: Identifier): number[] {
  // 检查市场数据字段
  if (this.context.isMarketDataField(node.name)) {
    return this.context.getMarketDataField(node.name);
  }

  // 检查变量
  if (this.context.hasVariable(node.name)) {
    return this.context.getVariable(node.name)!;
  }

  throw new Error(`Undefined identifier: ${node.name}`);
}

visitNumberLiteral(node: NumberLiteral): number[] {
  // 创建与市场数据长度相同的数组，所有值都是字面量值
  const length = this.context.getDataLength();
  return new Array(length).fill(node.value);
}
```

## 细节与备注

### 1. 数组操作模型

解释器采用向量化计算模型：
- 所有操作都是逐元素进行的
- 数组长度取最小长度（对于二元操作）
- 字面量自动扩展为数组

### 2. 类型系统

虽然 TypeScript 提供静态类型，但运行时：
- 所有数值都是 `number[]`
- 布尔值表示为 1（true）或 0（false）
- NaN 值正确传播

### 3. 函数参数处理

对于需要常量参数的函数（如 MA 的周期）：
```typescript
getConstantValue(arr: number[]): number {
  const firstValue = arr.find((v) => !isNaN(v));
  return firstValue !== undefined ? firstValue : arr[0];
}
```

### 4. 错误处理

- 未定义标识符会抛出明确的错误
- 未知函数会抛出错误
- 数学错误（如除零）返回 Infinity/NaN

### 5. 性能考虑

- 使用 Map 进行 O(1) 变量查找
- 避免不必要的数组复制
- 逐元素操作可以优化（未来工作）
