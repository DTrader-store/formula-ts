# Token 类型定义

**日期**: 2025-11-14

## 问题描述

需要实现词法分析器（Lexer）的基础 Token 系统，包括：
1. 定义所有需要的 Token 类型（enum）
2. 创建 Token 类来表示单个 Token 对象
3. 编写完整的单元测试验证实现

## 解决方案

按照 TDD 流程，采用以下步骤进行实现：

### 步骤 1: 创建 TokenType 枚举

在 `src/lexer/TokenType.ts` 中定义了 27 种 Token 类型，包括：
- **字面量**: NUMBER, IDENTIFIER
- **算术运算符**: PLUS, MINUS, MULTIPLY, DIVIDE
- **比较运算符**: GT, LT, GTE, LTE, EQ, NEQ
- **逻辑运算符**: AND, OR
- **标点符号**: LPAREN, RPAREN, COMMA, SEMICOLON, COLON
- **赋值**: ASSIGN
- **关键字**: IF
- **图表属性**: COLOR, LINETHICK, DOTLINE, STICK
- **特殊**: NEWLINE, EOF

### 步骤 2: 创建 Token 类

在 `src/lexer/Token.ts` 中实现 Token 类，具有以下特性：
- **属性**（只读）:
  - `type`: TokenType - Token 的类型
  - `value`: string - Token 的值
  - `line`: number - 所在行号（1-indexed）
  - `column`: number - 所在列号（1-indexed）
- **方法**:
  - `toString()`: 返回可读的字符串表示
  - `equals()`: 检查两个 Token 是否相等

### 步骤 3: 编写单元测试

在 `tests/unit/lexer/Token.test.ts` 中创建了 13 个单元测试，覆盖：
1. **构造函数测试**: 验证 Token 正确初始化，支持所有 Token 类型
2. **toString 方法测试**: 验证字符串表示格式，包括特殊字符处理
3. **equals 方法测试**: 验证 Token 相等性比较的所有场景
4. **枚举验证测试**: 验证 TokenType 包含所有必需的类型
5. **集成测试**: 验证多个 Token 的创建和比较

## 细节与备注

### 代码质量特点

1. **类型安全**: 使用 TypeScript 的 readonly 修饰符确保 Token 不可变性
2. **文档完整**: 所有类和方法都包含 JSDoc 注释
3. **测试覆盖**: 单元测试覆盖所有主要代码路径
4. **错误处理**: toString() 方法使用 JSON.stringify 正确处理特殊字符

### 设计决策

1. **Token 类使用 readonly 属性**: 这确保了 Token 一旦创建就不能被修改，符合不可变对象模式
2. **枚举值为字符串**: 便于调试和日志输出
3. **1-indexed 行列号**: 符合编辑器和错误报告的标准约定
4. **equals() 方法**: 用于测试和比较，比 === 更灵活

### 与后续模块的关系

- **Lexer 模块**: 本 Token 系统是 Lexer 的核心数据结构
- **Parser 模块**: Parser 会使用 Token 流进行语法分析
- **Error 处理**: 错误消息会使用 Token 的位置信息（line, column）
