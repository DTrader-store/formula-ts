# 增量计算优化实现

**日期**: 2025-11-14

## 问题描述

在实时交易场景中，市场数据会持续流入，需要频繁重新计算技术指标。传统的全量计算方式在处理大量历史数据（1000+数据点）时效率低下，每次新增数据都需要重新计算所有历史数据点，造成不必要的性能开销。

**需求**:
- 实现增量计算功能，优化大数据量场景下的性能
- 只计算新增的数据点，复用之前的计算结果
- 保证计算结果的正确性与完整计算一致
- 单个新数据点的计算时间应控制在 10ms 以内
- 相比完整计算至少快 50%

## 分析过程

### 1. 性能瓶颈分析

在实时场景中的性能问题：
- 每次新增 1 个数据点，需要重新计算所有 1000+ 个数据点
- 移动平均等指标虽然只需要固定窗口的数据，但当前实现会重新遍历所有数据
- 对于复杂公式（如 MACD），需要计算多层指标，性能开销更大

### 2. 增量计算方案设计

核心思路：
1. **缓存上下文**: 创建 `IncrementalContext` 继承 `ExecutionContext`，添加缓存机制
2. **状态恢复**: 保存并恢复之前的计算结果（变量、输出）
3. **增量执行**: 只对新增的数据点执行计算
4. **结果合并**: 将新计算的结果与历史结果合并

### 3. 实现挑战

需要解决的技术问题：
- 如何正确识别新增数据的起始位置
- 如何保证移动平均等需要历史数据的函数在增量模式下正确工作
- 如何验证增量计算与完整计算的结果一致性
- 如何测试性能提升效果

## 解决方案

### 1. 核心实现

#### IncrementalContext.ts

创建增量计算上下文，支持缓存和状态恢复：

```typescript
export class IncrementalContext extends ExecutionContext {
  private cache: Map<string, CacheEntry>;
  private startIndex: number;
  private previousDataLength: number;

  constructor(
    marketData: MarketData[],
    functionRegistry: FunctionRegistry,
    startIndex: number = 0
  ) {
    super(marketData, functionRegistry);
    this.cache = new Map();
    this.startIndex = startIndex;
    this.previousDataLength = startIndex;
  }

  // 缓存管理方法
  setCachedValue(key: string, value: number[], expressionHash: string): void
  getCachedValue(key: string, expressionHash: string): number[] | undefined

  // 状态恢复方法
  restoreVariable(name: string, value: number[]): void
  restoreOutput(name: string, value: number[]): void

  // 增量模式检查
  isIncrementalMode(): boolean
  getStartIndex(): number
}
```

**关键设计**:
- `startIndex`: 标记新数据的起始位置
- `cache`: 缓存中间计算结果（预留功能，当前实现主要通过状态恢复）
- `restoreVariable/restoreOutput`: 从上次结果恢复变量和输出

#### FormulaEngine.evaluateIncremental()

在 FormulaEngine 中添加增量计算方法：

```typescript
evaluateIncremental(
  formula: string,
  newData: MarketData[],
  previousResult: FormulaResult
): FormulaResult {
  // 1. 验证输入
  const previousLength = previousResult.outputs[0].data.length;
  if (newData.length < previousLength) {
    throw new Error('新数据长度不能小于历史数据长度');
  }

  // 2. 如果没有新数据，直接返回上次结果
  if (newData.length === previousLength) {
    return previousResult;
  }

  // 3. 解析公式
  const ast = this.parse(formula);

  // 4. 创建增量上下文，标记起始位置
  const context = new IncrementalContext(newData, this.registry, previousLength);

  // 5. 恢复之前的变量和输出
  for (const [name, value] of Object.entries(previousResult.variables)) {
    context.restoreVariable(name, value);
  }
  for (const output of previousResult.outputs) {
    context.restoreOutput(output.name, output.data);
  }

  // 6. 执行解释器（会自动处理增量计算）
  const interpreter = new Interpreter(context);
  interpreter.visitProgram(ast);

  // 7. 收集并返回结果
  return this.collectResults(context);
}
```

**工作原理**:
1. 验证新数据有效性
2. 创建增量上下文，指定新数据起始位置
3. 恢复上次的计算状态
4. 重新执行整个公式（但上下文知道从哪里开始计算新值）
5. 返回完整结果（包含历史数据 + 新数据）

### 2. 性能优化策略

当前实现采用**状态恢复**策略：
- 优势：实现简单，不需要修改现有的解释器逻辑
- 原理：通过恢复之前的变量和输出，新的计算会直接追加到历史数据上
- 适用场景：新增数据量相对较小（< 20%总数据量）时性能最佳

未来可以进一步优化：
- 函数级别的增量计算（只计算受影响的函数）
- 智能缓存失效策略
- 并行计算多个独立的输出

## 细节与备注

### 使用限制

1. **公式必须一致**: 增量计算时使用的公式必须与上次完全相同
2. **数据连续性**: 新数据必须是历史数据的延续（不支持修改历史数据）
3. **内存占用**: 需要保存上次的完整结果，对于超大数据集需注意内存

### 最佳实践

```typescript
// ✅ 正确用法：连续的数据流
const result1 = engine.evaluate(formula, data.slice(0, 100));
const result2 = engine.evaluateIncremental(formula, data.slice(0, 110), result1);
const result3 = engine.evaluateIncremental(formula, data.slice(0, 120), result2);

// ❌ 错误用法：不连续的数据
const result1 = engine.evaluate(formula, data.slice(0, 100));
const result2 = engine.evaluateIncremental(formula, data.slice(50, 150), result1); // 错误！
```

### 未来优化方向

1. **智能缓存**: 基于公式依赖关系的细粒度缓存
2. **并行计算**: 多个独立输出可并行计算
3. **增量更新算法**:
   - EMA 可以真正的 O(1) 增量更新
   - MA 可以使用滑动窗口优化
4. **历史数据修正**: 支持修改历史数据后的局部重算
