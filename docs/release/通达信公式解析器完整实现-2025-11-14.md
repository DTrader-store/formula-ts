# 通达信公式解析器完整实现

**日期**: 2025-11-14

## 问题描述

需要开发一个完整的通达信（TongDaXin）公式解析器和执行引擎，用于：

1. **解析通达信技术指标公式语法** - 支持变量声明、输出定义、运算符、函数调用、注释等完整语法
2. **执行公式计算** - 基于用户提供的市场数据，计算技术指标结果
3. **可视化展示** - 将计算结果输出为结构化数据，用于图表展示
4. **性能优化** - 支持增量计算，提高实时数据处理效率
5. **开发者工具** - 提供 VSCode 语法高亮扩展，提升开发体验

## 分析过程

### 需求调研

通过系统性的问答，明确了以下关键需求：

1. **使用场景**: 完整的解析 + 执行 + 可视化解决方案
2. **数据来源**: 用户自行提供市场数据（不内置数据获取）
3. **公式覆盖**: 分阶段实现，优先支持技术指标公式
4. **函数支持**: 可扩展的函数注册表，初期实现核心函数
5. **输出格式**: 结构化数据，包含绘图属性（颜色、线型、粗细等）
6. **图表库**: TradingView Lightweight Charts
7. **运行环境**: 同构设计（浏览器 + Node.js）
8. **错误处理**: 友好的错误信息，包含行列位置和修复建议
9. **性能要求**: 支持增量计算，优化实时数据更新场景
10. **语法支持**: 完整的通达信公式语法

### 架构设计

采用经典编译器架构：

```
公式字符串 → Lexer (词法分析) → Token 流
           → Parser (语法分析) → AST (抽象语法树)
           → Interpreter (解释执行) → 计算结果
```

**核心设计模式**:
- **Visitor 模式**: AST 遍历和执行
- **Registry 模式**: 函数管理和扩展
- **Strategy 模式**: 输出格式化
- **TDD 方法**: 测试驱动开发，确保质量

### 分阶段实施

将开发分为 3 个阶段（Phase），共 6 个冲刺（Sprint）：

**Phase 1 - MVP (Sprint 1-4)**:
- 项目基础设施
- Token 系统和词法分析器

#### Parser (语法分析器)
**文件**: `src/parser/Parser.ts` (507 行)

- 递归下降解析器
- 运算符优先级处理：逻辑运算 > 比较运算 > 算术运算
- 支持完整语法：变量声明 (`:=`)、输出定义 (`:`)、表达式、函数调用

**测试**: 59 个测试用例全部通过

