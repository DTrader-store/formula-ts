# 核心内置函数实现

**日期**: 2025-11-14

## 问题描述

需要实现 formula-ts 项目的 10 个核心内置函数，以支持公式解析器的完整功能。这些函数是技术分析公式的基础组件，包括：

- **数学函数**（5个）：MA、EMA、SUM、MAX、MIN
- **引用函数**（3个）：REF、HHV、LLV
- **逻辑函数**（2个）：IF、CROSS

所有函数需要支持时间序列数据（number[] 数组）操作，并需要完整的单元测试覆盖。

## 分析过程

### 1. 项目结构分析
检查了项目的现有结构：
- 已有 `FunctionRegistry` 类用于函数注册和管理
- 测试框架配置为 Jest + ts-jest
- 测试文件需要放置在 `tests/` 目录下

### 2. 函数设计原则
- 所有函数接受 `number[]` 数组作为输入
- 返回相同长度的 `number[]` 数组
- 对于历史数据不足的位置，返回 `NaN`
- 遵循 TDD 开发流程：先写测试，再实现功能

### 3. 技术规格确认
- **MA**: 简单移动平均，计算 N 周期的算术平均值
- **EMA**: 指数移动平均，使用 multiplier = 2/(N+1)，首值为 SMA
- **SUM**: N 周期滚动求和
- **MAX/MIN**: 元素级别的最大/最小值比较
- **REF**: 获取 N 周期前的值
- **HHV/LLV**: N 周期内的最高/最低值
- **IF**: 条件选择（非零为真）
- **CROSS**: 检测 A 上穿 B（前一个 A < B 且当前 A >= B）

#### 1. 实现文件（4个）

**src/interpreter/functions/math.ts**
- `MA(data, period)`: 简单移动平均

  - 前 N-1 个值返回 NaN
- `EMA(data, period)`: 指数移动平均

- `MAX(a, b)`: 元素级最大值

- `REF(data, period)`: 引用历史值
  - 简单的数组索引回溯
  - period = 0 返回原数组
- `HHV(data, period)`: 周期最高值
  - 滑动窗口内找最大值
- `LLV(data, period)`: 周期最低值
  - 滑动窗口内找最小值

**src/interpreter/functions/logical.ts**
- `IF(condition, a, b)`: 条件选择

### 代码示例

#### MA 函数实现
```typescript
export function MA(data: number[], period: number): number[] {
  const result: number[] = new Array(data.length);

  for (let i = 0; i < data.length; i++) {
    if (i < period - 1) {
      result[i] = NaN;
      continue;
    }

    let sum = 0;
    for (let j = 0; j < period; j++) {
      sum += data[i - j];
    }
    result[i] = sum / period;
  }

  return result;
}
```

#### CROSS 函数实现
```typescript
export function CROSS(a: number[], b: number[]): number[] {
  const length = Math.min(a.length, b.length);
  const result: number[] = new Array(length);

  result[0] = 0; // 首个位置无历史数据

  for (let i = 1; i < length; i++) {
    const wasBelowPreviously = a[i - 1] < b[i - 1];
    const isAboveOrEqualNow = a[i] >= b[i];
    result[i] = wasBelowPreviously && isAboveOrEqualNow ? 1 : 0;
  }

  return result;
}
```

## 细节与备注

### 技术亮点

1. **边界处理**
   - 所有函数正确处理历史数据不足的情况（返回 NaN）
   - 数组长度不一致时使用较短长度

2. **算法准确性**
   - EMA 使用标准公式 multiplier = 2/(N+1)
   - CROSS 精确检测上穿条件（需要前后两个时间点数据）

3. **测试覆盖**
   - 边界情况：period=0, period=1, period>length
   - 特殊值：相等值、全真、全假
   - 数组操作：不同长度数组处理

### 性能考虑

- **时间复杂度**:
  - MA, EMA, SUM: O(n*m) - n为数据长度，m为周期
  - MAX, MIN, IF, CROSS: O(n)
  - REF: O(n)
  - HHV, LLV: O(n*m)

- **空间复杂度**: 所有函数都是 O(n)，仅存储结果数组

### 未来改进方向

1. **性能优化**
   - 为 MA/SUM 实现滑动窗口优化（O(n)）
   - 为 HHV/LLV 使用单调队列（O(n)）

2. **功能扩展**
   - 添加更多技术指标函数（RSI, MACD, BOLL等）
   - 支持多维数据（OHLCV）

3. **集成工作**
   - 将函数集成到 Interpreter 中
   - 实现函数调用的运行时求值

4. **注意事项**
   - 当前的 `registerBuiltinFunctions` 实现是占位符
   - 实际的解释器集成需要处理数组与标量的转换
   - 需要在解释器层面统一管理数据长度
