## 问题描述

需要实现完整的Parser（语法分析器），将Token序列转换为抽象语法树（AST），支持通达信公式语言的完整语法，包括：
- 变量声明和输出声明
- 各种运算符（算术、比较、逻辑）及其优先级
- 函数调用
- 绘图属性解析

## 分析过程

### 1. 需求分析

通达信公式语法特点：
- **两种声明方式**：
  - 变量声明：`VAR1 := expr;` （中间变量，不显示）
  - 输出声明：`MA5: expr, COLORRED;` （输出线，在图表中显示）

- **运算符优先级**（从低到高）：
  1. 逻辑：AND, OR
  2. 比较：>, <, >=, <=, =, <>
  3. 加减：+, -
  4. 乘除：*, /
  5. 一元：-
  6. 函数调用和括号（最高优先级）

- **函数调用**：`MA(CLOSE, 5)`、`IF(cond, true_val, false_val)`

- **绘图属性**：`COLORRED`, `LINETHICK2`, `DOTLINE`, `STICK`

### 2. 设计决策

**采用递归下降解析**：
- 易于实现和理解
- 可以清晰地处理运算符优先级
- 错误处理直观

**优先级爬升方法**：
每个优先级层次一个解析方法：
```
parseExpression() -> parseLogicalOr()
parseLogicalOr() -> parseLogicalAnd()
parseLogicalAnd() -> parseComparison()
parseComparison() -> parseAddition()
parseAddition() -> parseMultiplication()
parseMultiplication() -> parseUnary()
parseUnary() -> parsePrimary()
```

**类型安全**：
- 显式声明返回类型
- 使用TypeScript接口定义AST节点
- 避免类型推断错误

### 2. Parser实现

**核心方法**：

1. **辅助方法**：
   ```typescript
   - check(type): 检查当前token类型
   - advance(): 前进到下一个token
   - consume(type, message): 消费期望的token
   - peek(): 查看当前token
   - peekNext(): 查看下一个token
   - error(message): 创建ParserError
   ```

2. **语句解析**：
   ```typescript
   - parseStatement(): 识别并解析变量/输出声明
   - parseVariableDeclaration(): VAR := expr;
   - parseOutputDeclaration(): NAME: expr, STYLE;
   - parseDrawingStyle(): 解析绘图属性
   ```

3. **表达式解析**（优先级递减）：
   ```typescript
   - parseExpression(): 入口方法
   - parseLogicalOr(): OR运算
   - parseLogicalAnd(): AND运算
   - parseComparison(): 比较运算
   - parseAddition(): 加减运算
   - parseMultiplication(): 乘除运算
   - parseUnary(): 一元运算
   - parsePrimary(): 基本表达式（数字、标识符、括号、函数）
   - parseFunctionCall(): 函数调用
   ```

```typescript
const binaryExpr: BinaryExpression = {
  type: NodeType.BinaryExpression,
  left,
  operator,
  right,
};
```

## 细节与备注

### 1. 运算符优先级实现

使用经典的优先级爬升（Precedence Climbing）方法，每个优先级一个方法，低优先级方法调用高优先级方法：

```
逻辑OR (最低) -> 逻辑AND -> 比较 -> 加减 -> 乘除 -> 一元 -> 基本 (最高)
```

这样自然实现了正确的优先级：
- `1 + 2 * 3` 被解析为 `1 + (2 * 3)`
- `A > B AND C < D` 被解析为 `(A > B) AND (C < D)`

### 2. TypeScript类型处理

最初遇到类型推断问题，对象字面量被推断为`Expression`而非具体类型。解决方法：
- 导入具体的AST接口类型
- 显式声明变量类型：`const binaryExpr: BinaryExpression = {...}`

### 3. 绘图属性映射

通达信绘图属性到内部表示的映射：
- `COLORRED` 等 -> `style.color`
- `LINETHICK1-9` -> `style.size` (提取数字)
- `DOTLINE` -> `style.italic = true`
- `STICK` -> `style.bold = true`

### 4. 函数调用特殊处理

IF关键字既是关键字又可作为函数名，需要在`parsePrimary()`中特殊处理：
```typescript
if (this.check(TokenType.IDENTIFIER) || this.check(TokenType.IF)) {
  // 同时支持普通函数和IF函数
}
```

### 5. 经典公式支持验证

测试了三个经典指标的完整解析：

**MA (移动平均线)**：
```
MA5: MA(CLOSE, 5), COLORWHITE;
MA10: MA(CLOSE, 10), COLORYELLOW;
MA20: MA(CLOSE, 20), COLORPINK;
```

**MACD**：
```
DIF: EMA(CLOSE, 12) - EMA(CLOSE, 26), COLORWHITE;
DEA: EMA(DIF, 9), COLORYELLOW;
MACD: (DIF - DEA) * 2, STICK;
```

**KDJ**：
```
RSV: (CLOSE - LLV(LOW, 9)) / (HHV(HIGH, 9) - LLV(LOW, 9)) * 100;
K: SMA(RSV, 3, 1), COLORWHITE;
D: SMA(K, 3, 1), COLORYELLOW;
J: 3 * K - 2 * D, COLORPINK;
```

所有公式都能正确解析为AST。
