# 错误类型定义实现报告

**日期**: 2025-11-14

## 问题描述

实现 Formula-TS 项目的错误处理系统，定义用于词法分析、语法分析和运行时执行阶段的错误类。这些错误类需要提供清晰的错误信息和位置信息（行号、列号），以帮助开发者快速定位和修复问题。

## 分析过程

### 1. 需求分析

根据 TDD（测试驱动开发）流程，需要实现以下错误类：
- `FormulaError`: 基础错误类（继承 Error）
- `LexerError`: 词法分析阶段错误，包含行号、列号和可选的字符信息
- `ParserError`: 语法分析阶段错误，包含行号和列号
- `RuntimeError`: 运行时错误

### 2. 实现策略

采用 TDD 流程：
1. **先写测试**: 创建全面的测试用例覆盖所有错误类的功能
2. **再写实现**: 实现错误类以满足测试要求
3. **验证通过**: 确保所有测试通过

### 3. 关键设计决策

- **继承结构**: 所有错误类继承自 `FormulaError`，而 `FormulaError` 继承自 JavaScript 原生 `Error`
- **错误消息格式**: 统一的格式便于日志解析和问题追踪
  - LexerError: `"Lexer error at line X, column Y: message (char: c)"`
  - ParserError: `"Parser error at line X, column Y: message"`
  - RuntimeError: `"Runtime error: message"`
- **原型链设置**: 使用 `Object.setPrototypeOf()` 确保 `instanceof` 检查和错误堆栈追踪正常工作

## 解决方案

### 1. 错误类实现

在 `/src/errors/index.ts` 中实现了四个错误类：

```typescript
export class FormulaError extends Error {
  constructor(message: string) {
    super(message);
    this.name = 'FormulaError';
    Object.setPrototypeOf(this, FormulaError.prototype);
  }
}

export class LexerError extends FormulaError {
  readonly line: number;
  readonly column: number;
  readonly char?: string;

  constructor(message: string, line: number, column: number, char?: string) {
    const charPart = char !== undefined ? ` (char: ${char})` : '';
    const fullMessage = `Lexer error at line ${line}, column ${column}: ${message}${charPart}`;
    super(fullMessage);
    this.name = 'LexerError';
    this.line = line;
    this.column = column;
    this.char = char;
    Object.setPrototypeOf(this, LexerError.prototype);
  }
}

export class ParserError extends FormulaError {
  readonly line: number;
  readonly column: number;

  constructor(message: string, line: number, column: number) {
    const fullMessage = `Parser error at line ${line}, column ${column}: ${message}`;
    super(fullMessage);
    this.name = 'ParserError';
    this.line = line;
    this.column = column;
    Object.setPrototypeOf(this, ParserError.prototype);
  }
}

export class RuntimeError extends FormulaError {
  constructor(message: string) {
    const fullMessage = `Runtime error: ${message}`;
    super(fullMessage);
    this.name = 'RuntimeError';
    Object.setPrototypeOf(this, RuntimeError.prototype);
  }
}
```

## 细节与备注

### 关键实现细节

1. **原型链修复**: 使用 `Object.setPrototypeOf()` 确保自定义错误类在 TypeScript 编译后仍能正确继承，这是处理 TypeScript 错误类的最佳实践。

2. **错误消息一致性**: 每个错误类都有统一的消息格式，便于：
   - 日志解析和分析
   - 用户界面显示
   - 调试和问题追踪

3. **可选字符信息**: LexerError 的 `char` 字段是可选的，允许灵活地提供导致错误的字符信息。

4. **只读属性**: 使用 `readonly` 关键字确保错误信息不被意外修改。

### 使用示例

```typescript
// LexerError 使用
throw new LexerError('Unexpected character', 1, 5, '$');
// 消息: "Lexer error at line 1, column 5: Unexpected character (char: $)"

// ParserError 使用
throw new ParserError('Expected closing parenthesis', 2, 10);
// 消息: "Parser error at line 2, column 10: Expected closing parenthesis"

// RuntimeError 使用
throw new RuntimeError('Division by zero');
// 消息: "Runtime error: Division by zero"

// 错误捕获
try {
  // 某个操作
} catch (error) {
  if (error instanceof LexerError) {
    console.error(`Line ${error.line}, Column ${error.column}: ${error.message}`);
  }
}
```
